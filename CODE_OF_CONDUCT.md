# Code of Conduct

NamiDB is a project by [Fonles Studios, Corp.](https://namidb.com). We
want it to be a place where engineers from anywhere on the planet feel
welcome to learn, contribute and disagree productively.

## Our standards

- **Be kind.** Assume good faith. People are doing their best with the
  context they have.
- **Be specific.** Disagree with code, designs, decisions — not with
  identities. Lead with what would change your mind.
- **Be respectful.** Personal attacks, harassment, and bigotry of any
  kind are not welcome.
- **Be patient.** Maintainers are part-time. New contributors take
  time to find their footing. Both directions.
- **Be honest.** Credit where it is due. Acknowledge mistakes. Update
  rather than delete when you change your mind.

## Scope

This Code of Conduct applies in every project space — GitHub issues,
pull requests, discussions, Discord (when launched), the commit log,
and any public event where you are representing NamiDB or Fonles
Studios, Corp.

## Enforcement

If you experience or witness behaviour that violates this Code of
Conduct, report it privately to
[`info@namidb.com`](mailto:info@namidb.com). All reports are read by
the maintainers, taken seriously, and handled confidentially.

Maintainers may, at their discretion, edit or remove comments,
commits, issues, pull requests, and other contributions that are not
aligned with this Code of Conduct, and may temporarily or permanently
ban any contributor for behaviours they deem inappropriate,
threatening or harmful.

## Maintainers

Maintainers are held to a higher standard, not a lower one. The same
rules apply, and we expect to be called out when we fall short.

---

*This Code of Conduct is adapted in spirit from the
[Contributor Covenant](https://www.contributor-covenant.org), kept
deliberately short.*
